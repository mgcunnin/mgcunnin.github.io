BEFORE UPLOADING

1. Put your six map files inside the images folder.
2. Use these exact filenames:
       mainmap.jpg
       detail1.jpg
       detail2.jpg
       detail3.jpg
       comparison1.jpg
       comparison2.jpg

UPLOADING TO GITHUB

Upload the CONTENTS of this folder to the top level of the existing
mgcunnin.github.io repository:

    images
    WAML
    index.html
    style.css
    START-HERE.txt

When GitHub asks whether to replace index.html and style.css, approve the
replacement. Commit directly to the main branch.

The old pragmatist-map folder can remain temporarily. Delete it only after
the new /WAML/ page is working.

EDITING THE ESSAY

Open WAML/index.html in GitHub and use the pencil icon.

Placeholder paragraphs have:
    class="placeholder"

You may replace the text while leaving the surrounding <p> and </p> tags.

When all placeholder text has been replaced, you may remove
class="placeholder" from the paragraph tags, although leaving it in place
will make the final prose slightly gray. It is better to remove it.

FOOTNOTES

One working linked footnote example is already included in the third
paragraph under “A Pragmatist Way of Reading Maps.”

To add another footnote marker in the prose, copy the pattern and change
all instances of the number:

    <sup class="footnote-ref" id="note-ref-2"><a href="#note-2" role="doc-noteref">2</a></sup>

Then add the return link to note 2:

    <a class="footnote-back" href="#note-ref-2" aria-label="Return to reference 2">↩</a>

SEARCH ENGINE VISIBILITY

WAML/index.html currently contains:

    <meta name="robots" content="noindex">

This asks search engines not to index the page while you are developing it.
Remove that line when you want the finished essay to be discoverable.
